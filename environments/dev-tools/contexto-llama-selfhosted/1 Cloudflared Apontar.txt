1. Cloudflared — Apontar URL para Open WebUI                                                                                                                                            
                                                                                                                                                                                          
  Como seu tunnel é gerenciado pelo dashboard do Cloudflare Zero Trust, você configura as rotas lá (não localmente). Vá em:                                                               

  Cloudflare Zero Trust Dashboard > Networks > Tunnels > selecione seu tunnel > Public Hostname                                                                                           
                                                                                                                                                                                          
  Adicione uma nova rota:                                                                                                                                                                 
  ┌───────────┬──────────────────────────────────┐                                                                                                                                        
  │   Campo   │              Valor               │                                                                                                                                        
  ├───────────┼──────────────────────────────────┤
  │ Subdomain │ chat (ou o que preferir)         │
  ├───────────┼──────────────────────────────────┤
  │ Domain    │ seu domínio (ex: seudominio.com) │
  ├───────────┼──────────────────────────────────┤
  │ Type      │ HTTP                             │
  ├───────────┼──────────────────────────────────┤
  │ URL       │ localhost:3000                   │
  └───────────┴──────────────────────────────────┘
  Configurações adicionais recomendadas (em Additional application settings):
  - TLS > No TLS Verify: ON (não precisa, é HTTP local)
  - HTTP Settings > HTTP Host Header: deixe em branco

  Depois de salvar, chat.seudominio.com vai apontar para o Open WebUI.

  2. n8n — Conectar ao Ollama

  No seu n8n, existem duas formas:

  Opção A: Nó nativo "Ollama" (Recomendado)

  No n8n, ao criar um workflow com nós de IA, use a credencial Ollama:
  ┌──────────┬─────────────────────────────┐
  │  Campo   │            Valor            │
  ├──────────┼─────────────────────────────┤
  │ Base URL │ http://192.168.68.115:11434 │
  ├──────────┼─────────────────────────────┤
  │ Model    │ llama3.1:8b-instruct-q4_K_M │
  └──────────┴─────────────────────────────┘
  Use o IP da máquina (192.168.68.115) porque o n8n e o Ollama estão em redes overlay diferentes no Swarm, e a porta está publicada em mode: host.

  Opção B: Nó "HTTP Request" (para controle total)
  ┌─────────────┬──────────────────────────────────────────┐
  │    Campo    │                  Valor                   │
  ├─────────────┼──────────────────────────────────────────┤
  │ Method      │ POST                                     │
  ├─────────────┼──────────────────────────────────────────┤
  │ URL         │ http://192.168.68.115:11434/api/generate │
  ├─────────────┼──────────────────────────────────────────┤
  │ Body (JSON) │ ver abaixo                               │
  └─────────────┴──────────────────────────────────────────┘
  {
    "model": "llama3.1:8b-instruct-q4_K_M",
    "prompt": "{{ $json.texto }}",
    "stream": false
  }

  Para chat com histórico, use /api/chat:

  {
    "model": "llama3.1:8b-instruct-q4_K_M",
    "messages": [
      {"role": "system", "content": "Você é um assistente útil."},
      {"role": "user", "content": "{{ $json.mensagem }}"}
    ],
    "stream": false
  }
